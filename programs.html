File I/O: Death Star Database

This Python program involves creating a login and access control system for the Death Star super-weapon. Users are required to register with their usernames and passwords, which are verified against a file called names_and_ranks.txt containing the names and ranks of all individuals on board the Death Star. The code also stores user information in a separate file. Once logged in, users can order a strike, cancel a strike, or mark a strike as completed based on their rank. The access controls are based on a hierarchy of ranks from lowest to highest.

The technical functions within the code include opening the names_and_ranks.txt file, parsing the data to create a dictionary with names and ranks, prompting the user to register, verifying user eligibility and registration status, prompting the user to login, verifying the user's credentials, and providing access to different functions based on the user's rank. The code also includes various helper functions to handle different types of actions and return the list of actions that a rank can do. The strike information is stored in a separate file, and the program automatically deletes strike information when a strike is marked as completed.

File I/O: Investigating a Chancellor

This Python program involves analyzing a set of 10,000 text files related to a former chancellor who may have illegally removed classified documents from her former office. The task is to analyze the files and produce a report for Republic Intelligence with the distribution of the files based on their classification level, year of publication, authors, and topics.

In the first part of the program, the task is to write a program that reads all the documents, determines the distribution of the classifications of the files found, and outputs the counts to a file called norida_files_report.txt. The report should include the total number of documents for each classification level, and the order of the classification level should be from lowest to highest. The code prints out a header for the report and defines a list of the four possible classification levels. It then reads the contents of each file and counts the number of files for each classification level. The code outputs the counts to a file called norida_files_report.txt.

The second part of the program involves adding additional functionality to the program from part A, which re-opens the same file and adds information about the years of publication. The third part adds information about the topics, and the fourth part adds information about the authors. The report generated at the end of the assignment includes all of this information.
